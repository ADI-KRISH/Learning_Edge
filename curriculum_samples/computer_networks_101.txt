# Chapter 1: Introduction to Computer Networks
A computer network is a collection of interconnected devices that share resources and exchange data. Networks are classified by their scale: a Local Area Network (LAN) covers a small geographic area like a building, a Wide Area Network (WAN) spans cities or countries, and the Internet is a global network of networks. The OSI (Open Systems Interconnection) model provides a conceptual framework with seven layers: Physical (raw bit transmission), Data Link (framing and MAC addressing), Network (routing via IP), Transport (end-to-end delivery via TCP/UDP), Session (connection management), Presentation (data encoding), and Application (HTTP, DNS, FTP). The TCP/IP model simplifies this into four layers: Network Access, Internet, Transport, and Application. Bandwidth refers to the maximum data transfer rate of a link, measured in bits per second. Latency is the time delay for data to travel from source to destination. Throughput is the actual data rate achieved in practice, always less than or equal to bandwidth.

# Chapter 2: IP Addressing and Routing
Every device on a network is assigned an IP address. IPv4 uses 32-bit addresses written in dotted-decimal notation (e.g., 192.168.1.1), providing approximately 4.3 billion unique addresses. IPv6 uses 128-bit addresses to solve IPv4 address exhaustion. Subnet masks divide an IP address into network and host portions. CIDR (Classless Inter-Domain Routing) notation like 192.168.1.0/24 specifies that the first 24 bits are the network prefix. NAT (Network Address Translation) allows multiple devices to share a single public IP address by mapping private IP addresses to the router's public IP. Routing is the process of forwarding packets from source to destination across multiple networks. Routers maintain routing tables populated by protocols like OSPF (Open Shortest Path First, link-state) and BGP (Border Gateway Protocol, path-vector used between ISPs). The Dijkstra algorithm underpins OSPF to find the shortest path in a weighted graph of network links.

# Chapter 3: Transport Layer Protocols — TCP and UDP
The Transport Layer is responsible for end-to-end communication between processes on different hosts. TCP (Transmission Control Protocol) provides reliable, ordered, and error-checked delivery. It establishes connections via a three-way handshake (SYN → SYN-ACK → ACK), uses sequence numbers to reorder out-of-order packets, and implements flow control via sliding window and congestion control via algorithms like Slow Start, Congestion Avoidance, and Fast Retransmit. UDP (User Datagram Protocol) is connectionless and provides no delivery guarantees, making it faster and suitable for latency-sensitive applications like video streaming, VoIP, and DNS lookups. Port numbers (0–65535) identify specific processes on a host. Well-known ports include 80 (HTTP), 443 (HTTPS), 22 (SSH), 53 (DNS), and 25 (SMTP). Multiplexing allows multiple application streams to share a single network connection using different port numbers.

# Chapter 4: Application Layer Protocols and Network Security
HTTP (HyperText Transfer Protocol) is a stateless request-response protocol for the web. HTTPS adds TLS/SSL encryption on top of HTTP to protect data in transit. DNS (Domain Name System) translates human-readable domain names (e.g., google.com) into IP addresses using a hierarchical distributed database of resolvers, root servers, TLD servers, and authoritative servers. DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses to devices joining a network. Common network attacks include: Denial of Service (DoS) — overwhelming a server with traffic; Man-in-the-Middle (MITM) — intercepting communications; ARP Spoofing — poisoning the ARP cache to redirect traffic; and SQL Injection at the application layer. Firewalls filter traffic based on rules. Intrusion Detection Systems (IDS) monitor for suspicious activity. VPNs (Virtual Private Networks) create encrypted tunnels over public networks to secure communication.
