# Chapter 1: Foundations of Machine Learning
Machine Learning (ML) is a subfield of artificial intelligence that enables systems to learn from data and improve their performance without being explicitly programmed. At its core, ML involves three key components: a model (a mathematical function mapping inputs to outputs), a loss function (which measures how wrong the model's predictions are), and an optimization algorithm (which adjusts the model's parameters to minimize the loss). The three main learning paradigms are supervised learning (training on labeled input-output pairs), unsupervised learning (finding hidden patterns in unlabeled data), and reinforcement learning (learning through reward signals from an environment). Bias and variance are two fundamental sources of error: high bias means the model is too simple and underfits the data, while high variance means the model is too complex and overfits. Regularization techniques like L1 (Lasso) and L2 (Ridge) penalize large weights to reduce overfitting. Cross-validation is used to estimate generalization performance by training and evaluating on different data splits.

# Chapter 2: Classical Supervised Learning Algorithms
Linear Regression models the relationship between input features and a continuous output by fitting a straight line (or hyperplane) that minimizes the Mean Squared Error (MSE). Logistic Regression extends this to binary classification by applying the sigmoid function to produce probabilities. Decision Trees partition the feature space recursively using information gain (entropy reduction) or the Gini impurity criterion. Random Forests are ensembles of decision trees trained on random subsets of data and features, reducing variance through averaging. Gradient Boosting methods like XGBoost and LightGBM build trees sequentially, each one correcting the errors of the previous. Support Vector Machines (SVMs) find the maximum-margin hyperplane separating classes, and with the kernel trick they can handle non-linearly separable data by mapping inputs to higher-dimensional spaces. K-Nearest Neighbors (KNN) classifies a point based on the majority class of its k closest training examples in feature space.

# Chapter 3: Neural Networks and Deep Learning
Artificial Neural Networks (ANNs) are composed of layers of interconnected neurons. Each neuron computes a weighted sum of its inputs and passes it through a non-linear activation function such as ReLU (Rectified Linear Unit), sigmoid, or tanh. The Universal Approximation Theorem states that a sufficiently wide single hidden layer network can approximate any continuous function. Deep networks (many layers) learn hierarchical feature representations: early layers detect low-level features (edges, textures) while later layers detect high-level concepts (objects, faces). Backpropagation computes gradients of the loss with respect to all weights using the chain rule, enabling gradient descent optimization. Batch Normalization normalizes layer activations to stabilize training. Dropout randomly zeroes out neurons during training, acting as regularization. Convolutional Neural Networks (CNNs) use shared filter weights and spatial pooling to process grid-like data such as images efficiently.

# Chapter 4: Model Evaluation and Deployment
Proper model evaluation requires understanding key metrics. For classification: Accuracy, Precision (TP/(TP+FP)), Recall (TP/(TP+FN)), F1-Score (harmonic mean of precision and recall), and AUC-ROC (area under the receiver operating characteristic curve). For regression: Mean Absolute Error (MAE), Mean Squared Error (MSE), and Root Mean Squared Error (RMSE). The confusion matrix summarizes classification results across all classes. Hyperparameter tuning methods include Grid Search (exhaustive), Random Search (probabilistic), and Bayesian Optimization (surrogate model-based). Model deployment involves wrapping the trained model in an API (e.g., using FastAPI or Flask), containerizing with Docker, and serving at scale using platforms like Kubernetes or cloud ML services. Concept drift — when the real-world data distribution shifts over time — requires continuous monitoring and periodic retraining to maintain model performance.
